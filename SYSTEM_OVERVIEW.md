# 🏗️ DineFlow System Overview

This document provides a simple breakdown of the technologies used in the DineFlow Restaurant Management SaaS and what purpose they serve.

---

## 🛠️ Core Technology Stack

| Layer | Technology | Purpose |
| :--- | :--- | :--- |
| **Framework** | **Next.js 14** | The "engine" that runs the app. Handles both the front-end (UI) and back-end (API routes). |
| **Language** | **TypeScript** | Enhances JavaScript with types to prevent bugs and make the code easier to maintain. |
| **Database ORM** | **Prisma** | A tool that lets developers interact with the database using simple code instead of complex SQL queries. |
| **Database** | **SQLite / PostgreSQL** | Where all your data (hotels, orders, menu items) is stored. (Currently running on SQLite for local ease). |

---

## 🎨 Design & Interaction

| Area | Technology | Purpose |
| :--- | :--- | :--- |
| **Styling** | **Tailwind CSS** | A modern CSS framework for fast, custom UI design (colors, layouts, spacing). |
| **Animations** | **Framer Motion** | Used for smooth transitions, sliding menus, and button pops (Zomato-style feel). |
| **Icons** | **Lucide React** | A library of clean, professional icons used throughout the waiter and billing panels. |

---

## ⚙️ Logic & State Management

| Feature | Technology | Purpose |
| :--- | :--- | :--- |
| **Global State** | **Zustand** | Manages the "shopping cart" of the waiter panel and UI states across the app. |
| **Data Fetching** | **React Query** | Handles fetching data from the server, caching it, and refreshing it automatically. |
| **Forms** | **React Hook Form** | Used for managing input fields in checkout and settings screens. |
| **Validation** | **Zod** | Ensures that data entered (like phone numbers or prices) is in the correct format. |

---

## 🔐 Security & Utilities

| Feature | Technology | Purpose |
| :--- | :--- | :--- |
| **Authentication** | **NextAuth.js** | Manages user logins (Waiters, Admin, Kitchen) and session security. |
| **Password Security**| **BcryptJS** | Hashes (scrambles) passwords so they are stored safely in the database. |
| **Billing** | **jsPDF** | Automatically generates professional PDF receipts for customers when they pay. |
| **HTTP Client** | **Axios** | Used to make requests to external services or internal APIs. |

---

## 📂 Key Folders

- `/app`: Contains all the pages (Waiter Dashboard, Kitchen View, Billing).
- `/src/components`: The building blocks of the UI (buttons, cards, modals).
- `/prisma`: The blueprint for your database.
- `/public`: Static images and icons.
